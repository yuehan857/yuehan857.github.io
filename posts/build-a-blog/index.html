HW1: How I build this blog

Aug 22, 2020 · 3 min read

I would like to record the process of creating this blog. It is quite simple to use Hugo, a static site generator, to build ones own website! There are mainly two parts: build my website locally, and then deploy it on GitHub Pages.

Build website locally

  1. Open terminal, install hugo using brew.
brew install hugo
  1. Using hugo new to creat a new website, which is this blog.
hugo new site <SiteName>
  1. Download a theme using git clone.
cd <SiteName>/themes
git clone <ThemeURL>

Then apply this theme to website, just open the config.toml file in <SiteName> folder and change theme to <ThemeName>.

  1. Custmize website by editing config.toml file, e.g. change themeColor, add profile picture by adding a picture with .png format in <SiteName>/static/images folder, etc.

  2. Add new post using hugo new (need to cd back to <SiteName> folder).

cd ~<UserName>/<SiteName>
hugo new posts/<PostName.md>
  1. Almost done. Before launching the website locally, check if any post is in draft status (if so, delete that line, otherwise the post won’t be shown on website). Then run hugo server and check website locally on http://localhost:1313. Press Ctrl+C to quit.

Deploy website on GitHub

  1. On GitHub, create a repo named <GitHubUserName>.github.io to host website. It is important to use <GitHubUserName>, otherwise problems may occur. Click Settings on the right upper corner, roll to GitHub Pages section and choose master/root as source.

  2. Open config.toml file in <SiteName> folder, change baseURL to https://<GitHubUserName>.github.io/.

  3. Run hugo on terminal, and a public folder will be generated. It contains all the content needed to deploy a website. Initialize a local repo in public folder.

hugo
cd public
git init
  1. Commit changes and push to <GitHubUserName>.github.io repo. For the first time, set remote repo’s URL (which is <GitHubUserName>.github.io repo’s URL) and name it.
git remote add <Name> <RemoteRepoURL>
git add -A 
git commit -m"<message>"
git push <Name> master
  1. Any update for the website can be pushed to <GitHubUserName>.github.io repo in the same way as 3&4 (ignore git init and git remote add <Name> <RemoteRepoURL>). For convenience, I create a shell script named deploy.sh to automate this process. To make it executable , cd to the directory deploy.sh located and run chmod +x ./deploy.sh. Run ./deploy.sh, deploy will be completed automatically. Note it is not deploy.sh but ./deploy.sh. The code in deploy.sh is as following.
#!/bin/sh

# If an error occurs, stop.
set -e
printf "deploy fails"

# Update public folder.
cd ~<UserName>/<SiteName>
hugo

# Go to public folder.
cd public

# Add changes to git.
git add -A

# Commit changes.
git commit -m"update on $(date +%F)"

# Push to remote repo.
git push origin master
